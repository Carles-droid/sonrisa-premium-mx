# COMPORTAMIENTO POR BREAKPOINT

## Desktop (≥1024px)

- `NavbarMenu`: horizontal
- `Tratamientos`:
    - Hover → abre dropdown
    - Click opcional (fallback accesible)
- `LangToggle`: visible
- `CTA`: visible
- `Hamburger`: oculto

---

## Tablet (768px – 1023px)

- Igual que desktop, con ajuste de spacing
- Dropdown:
    - Preferible **click + hover híbrido**
- Mantener todo visible (sin colapsar aún)

---

## Mobile (<768px)

- `NavbarMenu`: oculto por defecto
- `NavbarToggle`: abre menú vertical

### Comportamiento clave:

- `Tratamientos`:
    - SOLO click
    - Expande tipo accordion

```
Tratamientos
  ↓
[ Limpieza ]
[ Blanqueamiento ]
[ Ortodoncia ]
...
```

- `LangToggle`: dentro de `NavbarActions` (visible en menú desplegado)
- `CTA`: visible dentro del menú + opcional sticky

# DECISIONES IMPORTANTES

### 1. `Tratamientos` usa `<button>` (no `<a>`)

Correcto:

- Evita navegación accidental
- Permite control total del dropdown

# HERO CON IMAGE SLIDER

## Estructura

```
Hero
├── HeroSlider
│   ├── SlideItem
│   │   ├── SlideBackgroundImage
│   │   └── SlideOverlay
│   │
│   ├── SlideItem (...)
│   └── SliderControls
│       ├── PrevButton
│       ├── NextButton
│       └── Indicators
│
├── HeroContent
│   ├── HeroHeadline
│   ├── HeroSubheadline
│   ├── HeroCtaGroup
│   │   ├── CtaPrimary (Agendar)
│   │   └── CtaSecondary (Ver Tratamientos)
│   └── HeroMeta (horarios / ubicación breve)
```

---

## Características técnicas

- `HeroContent` va en **overlay absoluto**
- `SlideItem` controla imagen (no `<img>`, usar background)

## Importancia en el proyecto

- Necesitas mostrar **varios servicios visualmente**
- Portafolio fuerte (antes/después, instalaciones, equipo)

# CONTENIDO MÍNIMO REQUERIDO

```
HeroHeadline:
→ Propuesta de valor clara (ej: “Sonrisas saludables con tecnología de vanguardia”)

HeroSubheadline:
→ Diferenciador (rapidez, especialistas, ubicación)

CtaPrimary:
→ Acción directa (Agendar)

CtaSecondary:
→ Acción exploratoria (Servicios / WhatsApp)

HeroMeta:
→ Datos rápidos (CDMX, horarios, urgencias)
```

# REGLAS DE LAYOUT

## 1) Separación de capas

```
Layer 1 → Slider (background dinámico)
Layer 2 → Overlay (oscurecimiento / contraste)
Layer 3 → HeroContent (interacción)
Layer 4 → Controls (navegación)
```

---

## 2) Jerarquía visual

1. Headline
2. CTA Primary
3. Subheadline
4. CTA Secondary
5. Meta

---

## 3) Constraints

- Máximo:
    - 2 CTAs
    - 1 mensaje principal
- No mezclar contenido por slide (el contenido es global, no por imagen)
